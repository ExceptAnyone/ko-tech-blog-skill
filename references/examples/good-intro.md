# 좋은 서론 예시

## 예시 1: 질문형 + 공감 유도

```markdown
"배포할 때마다 왜 이렇게 오래 걸릴까?"

개발자라면 누구나 한 번쯤 겪어봤을 거예요.
코드를 수정하고, 커밋하고, 푸시하고, 배포하는데...
커피 한 잔 마실 시간이 필요하죠.

우리 팀도 그랬어요. 배포 한 번 하는 데 10분이 걸렸거든요.
```

**왜 좋은가:**
- 질문으로 시작해 독자의 주의 환기
- "개발자라면 누구나"로 공감 유도
- 구체적인 시간(10분)으로 문제 규모 제시
- 일상적 언어(커피 한 잔) 사용

---

## 예시 2: 구체적 상황 묘사

```markdown
월요일 아침 9시, 슬랙 알림이 울렸어요.

"사이트가 느려요!"

커피를 들고 모니터를 확인했어요. 응답 시간이 3초...
새벽에는 0.5초였는데, 출근 시간대만 되면 느려지는 거예요.
```

**왜 좋은가:**
- 구체적인 시간과 상황(월요일 아침 9시)
- 현장감 있는 묘사(슬랙 알림, 커피)
- 직접 인용("사이트가 느려요!")
- 문제의 패턴 제시(출근 시간대)

---

## 예시 3: 감정 표현 + 문제 제시

```markdown
"또 빌드 실패야?"

프론트엔드 팀원들의 한숨이 깊어졌어요.
배포 한 번 하려면 빌드부터 다시 시작해야 했거든요.

작은 오타 하나 고치는 데도 10분씩 걸렸죠.
빠른 피드백이 중요한 스타트업 환경에서는 치명적이었어요.
```

**왜 좋은가:**
- 감정 표현(한숨이 깊어졌어요)
- 현장 대사 직접 인용
- 구체적인 예시(작은 오타 하나)
- 비즈니스 맥락 제시(스타트업 환경)

---

## 예시 4: "누구나" 패턴

```markdown
개발자라면 한 번쯤 이런 생각을 해봤을 거예요.

"이 코드, 왜 이렇게 복잡하지?"
"더 간단하게 만들 수 없을까?"

저도 그랬어요. 레거시 코드를 보며 고민했죠.
하지만 리팩토링할 시간은 없었고, 버그는 계속 생겼어요.
```

**왜 좋은가:**
- "누구나"로 보편성 강조
- 질문 2개로 공감대 형성
- 개인 경험과 연결
- 딜레마 제시(시간 vs 버그)

---

## 예시 5: 숫자로 임팩트 전달

```markdown
하루에 **100만 번**.

우리 서버가 동일한 상품 정보를 데이터베이스에서 조회하는 횟수예요.
사용자가 페이지를 열 때마다, 똑같은 쿼리가 실행되고 있었죠.

"이거 캐싱하면 안 될까?"

단순한 질문이었지만, 생각보다 복잡한 문제였어요.
```

**왜 좋은가:**
- 강력한 숫자로 시작 (100만 번)
- 문제의 규모 즉시 파악 가능
- 질문으로 해결책 암시
- "생각보다 복잡한"으로 호기심 유발

---

## 좋은 서론의 공통점

### 1. 독자 공감
- "누구나", "한 번쯤", "개발자라면"
- 일상적 경험 환기

### 2. 구체성
- 시간, 장소, 상황 명시
- 구체적인 숫자 제시

### 3. 감정 표현
- 한숨, 고민, 좌절
- 현장감 있는 묘사

### 4. 문제 암시
- 해결할 문제를 슬쩍 제시
- 하지만 너무 자세히 설명 X (다음 섹션에서)

### 5. 짧고 간결
- 3-5개 문단
- 각 문단은 1-3줄
- 너무 길면 독자 이탈

## 피해야 할 서론

### 나쁜 예시 1: 기술 용어 남발
```markdown
이번 글에서는 Incremental Static Regeneration과
On-Demand Revalidation을 활용한 Next.js 최적화 기법에 대해
다루겠습니다.
```

**문제점:**
- 기술 용어만 나열
- 공감 요소 없음
- 왜 중요한지 설명 없음
- 딱딱한 논문 스타일

---

### 나쁜 예시 2: 추상적 표현
```markdown
최근 웹 개발 트렌드는 성능 최적화에 집중하고 있습니다.
많은 개발자들이 이에 관심을 가지고 있죠.
```

**문제점:**
- 추상적이고 일반적
- 구체적 문제 없음
- "최근", "많은" 같은 모호한 표현
- 독자의 관심을 끌지 못함

---

### 나쁜 예시 3: 결론부터 말하기
```markdown
우리는 Redis를 도입해서 성능을 10배 개선했습니다.
이제 그 과정을 설명하겠습니다.
```

**문제점:**
- 긴장감 없음
- 결론을 미리 말해버림
- 독자가 읽을 이유 감소
- 스토리텔링 부재
